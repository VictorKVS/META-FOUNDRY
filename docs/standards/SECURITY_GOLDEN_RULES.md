# Golden Security Rules — META-FOUNDRY
Документ: MF-ST-04.01
Версия: 0.1.0 (черновик)


## Golden Rule #1 — Zero Secrets in Repo
Никаких ключей, токенов, .env, сертификатов.


## Golden Rule #2 — Security-by-Default
Безопасность должна быть встроена в код и процессы.


## Golden Rule #3 — Immutable History
История Git не подлежит изменению.


## Golden Rule #4 — Dual Control
Все изменения проходят review владельца.


## Golden Rule #5 — Automated Compliance
CI/CD валидирует всё автоматически.


## Golden Rule #6 — Least-Privilege Access
Минимизация доступа во всех процессах.


## Golden Rule #7 — AutoDoc-Driven Governance
Документация генерируется автоматически.


## Golden Rule #8 — Architecture Integrity Lock
Архитектура изменяется только через строгие правила.
